# Events

triangle-rs uses a broadcast `EventEmitter` to route all incoming ribbon messages and internal library events. Every event has a string name and a JSON payload.

## Typed event system

All library-defined events implement the `TypedEvent` trait which carries a compile-time `NAME` constant and can be serialised/deserialised as JSON.

```rust
pub trait TypedEvent {
    const NAME: &'static str;
}
```

### Listening (typed)

```rust
// Persistent тАФ fires for every matching event.
// Returns a JoinHandle; dropping it cancels the listener.
client.ribbon.emitter.on_typed::<MyEvent, _>(|ev: MyEvent| {
    println!("{ev:?}");
});

// One-shot async wait.
let ev: MyEvent = client.ribbon.emitter.once_typed::<MyEvent>().await.unwrap();
```

### Emitting (typed)

```rust
client.ribbon.emitter.emit_typed(&MyEvent { тАж });
```

### Untyped fallback

```rust
// Raw Value callback.
client.on("some.event", |data: Value| { тАж });

// Await next raw occurrence.
let data: Value = client.wait("some.event").await.unwrap();
```

## Client events

These are emitted by the library, not the ribbon server.

### `client.ready`

Fires when the ribbon handshake completes and `server.authorize` succeeds.

```rust
pub struct ClientReady {
    pub endpoint: String,
    pub social: Value,   // raw social init blob
}
```

### `client.fail`

The ribbon connection failed or the server rejected the token.

```rust
pub struct ClientFail {
    pub message: String,
}
```

### `client.error` тАФ `ClientError(String)`

A non-fatal ribbon error (e.g., command rejected by the server).

### `client.dead` тАФ `ClientDead(String)`

The ribbon task has terminated. The reason string is the last disconnect/error message.

### `client.close` тАФ `ClientClose(String)`

The connection was cleanly closed.

### `client.notify` тАФ `ClientNotify`

A user-visible notification generated by the library or server. Fields are optional styling hints for a UI.

```rust
pub struct ClientNotify {
    pub msg: String,
    pub timeout: Option<u32>,
    pub color: Option<String>,
    pub bgcolor: Option<String>,
    pub icon: Option<String>,
    pub header: Option<String>,
    // тАж
}
```

### `client.ribbon.receive` / `client.ribbon.send`

Raw ribbon traffic, useful for debugging.

```rust
pub struct ClientRibbonReceive { pub command: String, pub data: Value }
pub struct ClientRibbonSend    { pub command: String, pub data: Value }
```

## Game lifecycle events

### `client.game.start` тАФ `ClientGameStart`

A match has begun (before the countdown).

```rust
pub struct ClientGameStart {
    pub multi: bool,
    pub ft: u32,   // first-to (wins needed)
    pub wb: u32,   // win-by
    pub players: Vec<ClientGamePlayer>,
}

pub struct ClientGamePlayer {
    pub id: String,
    pub name: String,
    pub points: u32,
}
```

### `client.game.round.start` тАФ `ClientGameRoundStart`

A single round is starting. This event is **not** Serialize/Deserialize тАФ subscribe via `on`:

```rust
client.on("client.game.round.start", move |_| {
    // Use game.on_round_start instead (preferred):
});
```

Or, preferably, use `Game::on_round_start` which gives you a `TickSetter` directly тАФ see [Game.md](Game.md).

```rust
pub struct ClientGameRoundStart {
    pub setter: TickSetter,          // call setter.set(|input| async { тАж })
    pub engine: Arc<Mutex<Engine>>,  // read-only engine snapshot
}
```

### `client.game.over` тАФ `ClientGameOver`

The client's own game has ended.

```rust
pub struct ClientGameOver {
    pub reason: GameOverKind,
    pub data: Option<GameReplayEndData>,
}

pub enum GameOverKind { Finish, Abort, End, Leave }
```

### `client.game.round.end` тАФ `ClientGameRoundEnd(Option<String>)`

A round ended. Contains the winning player's ID (if any).

### `client.game.end` тАФ `ClientGameEnd`

The whole match has concluded.

```rust
pub struct ClientGameEnd {
    pub duration: f64,
    pub source: String,
    pub players: Vec<ClientGameEndPlayer>,
}

pub struct ClientGameEndPlayer {
    pub id: String,
    pub name: String,
    pub points: Option<u32>,
    pub won: bool,
    pub lifetime: Option<i64>,
}
```

### `client.game.abort` тАФ `ClientGameAbort`

The game was aborted before it completed.

## Raw ribbon events (game.\*)

These are emitted by the ribbon server and re-broadcast by the library. You normally don't need to listen to them directly тАФ the `Game` struct handles them internally.

| Name                | Struct              | Description                              |
| ------------------- | ------------------- | ---------------------------------------- |
| `game.ready`        | `GameReady`         | Players and options for an upcoming game |
| `game.start`        | `GameStart`         | Round is starting (after countdown)      |
| `game.abort`        | `GameAbort`         | Game aborted by server                   |
| `game.match`        | `GameMatch`         | Match metadata                           |
| `game.advance`      | `GameAdvance`       | Scoreboard update between rounds         |
| `game.score`        | `GameScore`         | Round score                              |
| `game.end`          | `GameEnd`           | Full match result and leaderboard        |
| `game.replay`       | `GameReplayInbound` | Replay frames from a spectated player    |
| `game.replay.state` | `GameReplayState`   | Full engine snapshot for spectation      |

```rust
pub struct GameReady {
    pub players: Vec<GameReadyPlayer>,
    pub is_new: bool,
}

pub struct GameReadyPlayer {
    pub gameid: u32,
    pub userid: String,
    pub options: Options,
    pub alive: bool,
    pub naturalorder: i32,
}

pub struct GameEnd {
    pub leaderboard: Vec<Leaderboard>,
    pub scoreboard: Vec<Scoreboard>,
    pub xp_per_user: f64,
    pub winners: Vec<Value>,
}
```
